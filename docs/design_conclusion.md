# 芦笋分拣视觉系统：设计结论

> **本文档是规划软件的设计基础，具有最终决策效力。**
> 所有下游软件架构、坐标系定义、标定流程均以本文档为准。

---

## 核心结论

**相机方案锁定：双目立体相机，固定安装于大地（Eye-to-Hand）。**

相机安装在独立支架上，物理固定于地面，与分拣机及机械臂的运动完全解耦。
相机俯视传送带工作区，坐标系固定不变，标定结果长期有效。

## 一、坐标系定义（规划软件的物理基准）

```
            相机光心（固定于大地，不动）
                    │
                    │  工作距离 Z_work ≈ 800 mm（待标定确认）
                    │
     ───────────────┼─────────────────── 传送带平面（Z = 0 基准面）
                    │
         芦笋堆放于此平面，高度 H ∈ [0, ~40 mm]
```

- **世界坐标系原点**：双目相机光心正下方在传送带平面的投影点。
- **X 轴**：沿传送带方向（芦笋长度方向，头朝 X+）。
- **Y 轴**：垂直于传送带，水平向右（拨钩滑台运动方向）。
- **Z 轴**：垂直向上，Z = 0 为传送带平面，Z > 0 为芦笋高度方向。

> **[!IMPORTANT]**
> 规划软件中所有目标点、拨钩轨迹、障碍物描述，均在此世界坐标系下表达。
> 相机输出的深度图经过一次固定的外参变换即可转化为世界坐标。

## 二、大地固定双目的物理优势（为何选此方案）

### 2.1 测距精度分析

双目视觉深度误差公式：

$$\Delta Z = \frac{Z^2}{B \cdot f_{\text{pixel}}} \cdot \Delta d$$

在大地固定方案中，工作距离 $Z \approx 800\text{ mm}$，选用基线 $B = 120\text{ mm}$，
$f_{\text{pixel}} \approx 1066\text{ px}$（1280px 宽，4mm 焦距，1/3'' 传感器），$\Delta d = 0.5\text{ px}$：

$$\Delta Z = \frac{800^2}{120 \times 1066} \times 0.5 \approx \pm 2.5\text{ mm}$$

| 指标 | 数值 |
|:---|:---|
| 深度误差（1σ） | ≤ ±2.5 mm |
| 芦笋高度分层精度 | 可区分 ≥ 5 mm 高度差（叠放层间距） |
| 对 4 mm 细芦笋的满足度 | **满足**：可判断上下叠加关系，不要求亚毫米级绝对精度 |

> 规划软件**不依赖**深度图中 4 mm 芦笋的精确绝对高度，
> 只需判断芦笋之间的**相对高度差**（哪根在上面），精度需求降至 ±3 mm 即满足。

### 2.2 标定稳定性

- 相机静止，外参矩阵永久有效，无需频繁重标定。
- 滑台启停产生的震动对相机**零影响**。
- 标定漂移风险：**不存在**。

### 2.3 遮挡问题的重新评估

大地固定方案的遮挡问题在传统分析中被列为"严重"，但在本系统中：

| 遮挡场景 | 实际影响 | 应对策略 |
|:---|:---|:---|
| 拨钩机构进入视野 | **可接受**：拨钩进入时，规划已完成，不需要实时感知 | 感知与执行分时进行 |
| 滑台机械壳体遮挡 | **局部遮挡**：相机高位俯视可规避结构遮挡 | 支架高度和角度标定时调整 |
| 芦笋自遮挡（上层挡下层边缘） | **同臂载方案相同**：本质是双目视角问题 | 采用"2D 定 X，3D 定 Z"策略 |

**关键洞察**：感知（拍照识别）与执行（拨钩动作）是分时进行的。
拍照时拨钩处于收起状态，无遮挡；执行时无需再拍照。遮挡问题**不成立**。

## 三、规划软件的输入接口定义

视觉系统向规划模块输出以下结构化数据：

```python
@dataclass
class AsparagusPose:
    """单根芦笋的三维位姿描述（世界坐标系）"""
    x_center: float      # 芦笋中心 X 坐标 [mm]，沿传送带方向
    y_center: float      # 芦笋中心 Y 坐标 [mm]，垂直传送带方向
    z_top: float         # 芦笋顶面高度 [mm]，Z=0 为传送带平面
    angle_deg: float     # 芦笋长轴与 X 轴的夹角 [°]，范围 ±15°
    diameter_mm: float   # 直径估计 [mm]，范围 4~40 mm
    layer: int           # 叠放层级：0=最底层（直接接触传送带），1=压在其他芦笋之上

@dataclass
class SceneSnapshot:
    """单次拍照识别结果"""
    timestamp_ms: int               # 拍照时刻（系统时间，毫秒）
    conveyor_position_mm: float     # 拍照时传送带的绝对位置 [mm]（由编码器提供）
    asparagus_list: list[AsparagusPose]  # 当前视野内所有可见芦笋
    rightmost_target: AsparagusPose | None  # 最右侧待拨出目标（视觉模块负责识别）
```

## 四、拨钩规划的坐标转换链

```
双目深度图（像素坐标）
    │
    │  [固定外参矩阵 T_cam2world，标定后不变]
    ▼
世界坐标系三维点云 (X, Y, Z) [mm]
    │
    │  [识别算法：芦笋分割 + 层级判断]
    ▼
AsparagusPose 列表
    │
    │  [规划模块：选目标 + 计算拨钩插入点]
    ▼
拨钩目标点 (Y_hook, Z_hook) [mm]，世界坐标系
    │
    │  [固定偏置补偿 d_cam_offset，一次性标定]
    ▼
滑台步进电机目标步数 + 舵机旋转角度
```

## 五、标定清单（大地固定方案）

| 标定项 | 方法 | 频率 |
|:---|:---|:---|
| **双目内参标定**（畸变系数、焦距） | 棋盘格，OpenCV stereoCalibrate | 出厂一次，相机不拆不重标 |
| **双目外参标定**（基线 B，旋转矩阵 R） | 同上，立体标定 | 同上 |
| **相机-世界外参标定** | 在传送带平面放置已知坐标标定板，求解 T_cam2world | 安装/移机后各一次 |
| **相机-拨钩 Y 轴偏置 d_cam_offset** | 细标记物对齐法（见下方说明） | 同上，或怀疑偏差时 |
| **Z 轴视差-高度映射验证** | 已知高度垫块（5 个高度点）验证深度精度 | 同上 |

### d_cam_offset 标定步骤

1. 在传送带上放置细标记点（牙签端点）。
2. 驱动滑台，使拨钩尖端在 X 轴方向对齐该标记，记录步进位置 $Y_{\text{hook}}$。
3. 通过相机图像确认该标记点在图像中的像素坐标，反投影到世界坐标得到 $Y_{\text{cam\_measure}}$。
4. 偏置：$d_{\text{cam\_offset}} = Y_{\text{hook}} - Y_{\text{cam\_measure}}$，写入配置文件。
